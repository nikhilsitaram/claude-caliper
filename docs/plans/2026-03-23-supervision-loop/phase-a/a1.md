# A1: Validate sleep-based polling pattern

**Avoid:** Don't skip this step even if the pattern seems obvious. The design doc explicitly gates the entire plan on this validation. If any criterion fails, the architecture needs revisiting (per-cycle monitor subagent fallback from Key Decision 1).

## Steps

### Step 1: Dispatch a background agent that runs for ~30 seconds

Dispatch a simple background agent using `Agent` with `run_in_background: true`. The agent should:
- Print "STARTED" immediately
- Run `Bash("sleep 10")`
- Print "AFTER_SLEEP_1"
- Run `Bash("sleep 10")`
- Print "AFTER_SLEEP_2"
- Run `Bash("sleep 5")`
- Print "COMPLETED"

Capture the returned `task_id`.

### Step 2: Verify supervisor remains free after dispatch

Immediately after dispatch (without waiting), run a trivial `Bash("echo supervisor_alive")`. This confirms the supervisor is not blocked by the background agent.

### Step 3: First poll cycle — sleep + TaskOutput

Run `Bash("sleep 10")` to simulate a poll interval. Then call `TaskOutput(task_id, block: false)`.

**Verify:** The response includes `status` (should be `running`) and `output` (should contain at least "STARTED"). This confirms criteria (a) sleep returns control and (b) TaskOutput reads background agent state.

### Step 4: Second poll cycle — sleep + TaskOutput

Run `Bash("sleep 10")` again. Then call `TaskOutput(task_id, block: false)`.

**Verify:** The output now contains more text than the first poll (e.g., "AFTER_SLEEP_1" appears). This confirms criterion (c) the loop continues and shows incremental progress.

### Step 5: Wait for completion

Run `Bash("sleep 15")` to give the background agent time to finish. Then call `TaskOutput(task_id, block: false)`.

**Verify:** Status is `completed` and output contains "COMPLETED". This confirms criterion (d) the background agent completes independently.

### Step 6: Document results

Record pass/fail for each criterion:
- (a) `Bash("sleep 10")` returns control to the agent
- (b) `TaskOutput(task_id, block: false)` returns status and output
- (c) Second sleep+TaskOutput cycle executes (loop continues)
- (d) Background agent completes independently

If all pass, this task succeeds. If any fail, stop and escalate — the plan is blocked per design doc.
