### Step 2: Write the supervision loop replacement for steps 5-6

Replace current step 5 (synchronous dispatch) with:

```text
5. **Dispatch implementer in background:**
   - Use Agent with run_in_background: true (see ./implementer-prompt.md for prompt)
   - Capture task_id, initialize: prev_output_len=0, prev_head_sha=$(git rev-parse HEAD), no_progress_count=0

6. **Supervision loop (every {DISPATCHER_POLL_SECONDS}s):**
   a. Bash("sleep {DISPATCHER_POLL_SECONDS}")
   b. Read TaskOutput(task_id, block: false) → get status and output
   c. If status == completed → break to step 7
   d. Extract new_output = output[prev_output_len:]
   e. Check detection signals on new_output (see Detection Logic below)
   f. Check git log --format=%H -1 vs prev_head_sha
   g. Evaluate health:
      - Permission prompt pattern in new_output → stuck
      - Same error line 3+ consecutive times in new_output → stuck
      - No new commits AND no new output for 2 consecutive polls → stuck
      - Otherwise → healthy, update prev_output_len and prev_head_sha, reset no_progress_count
   h. If stuck → run intervention protocol (see below)

7. **After implementer completes:** run task review loop (existing steps 6a-6e)
```

### Step 3: Write the detection logic section

Add a `## Detection Logic` section after `## Your Process` (or inline within step 6). Content from the design doc:

**Permission blocks:** Match multiline patterns containing "Do you want to proceed" followed by numbered options (e.g., `1. Yes`, `2. Yes, and don't ask again`). Single keywords like "permission" or "denied" in isolation are not sufficient.

**Repeated errors:** Extract lines matching error patterns (`error:`, `Error:`, `failed:`, `FAILED`, `Traceback`, `panic:`). Require the same error line 3+ consecutive times (not scattered occurrences).

**No progress:** Compare current git HEAD hash and TaskOutput length against previous cycle. Both unchanged → no progress. Two consecutive no-progress cycles → stuck.

### Step 4: Write the intervention protocol section

Add intervention protocol after detection logic:

| Attempt | Action |
|---------|--------|
| 1st | TaskStop(task_id) + re-dispatch implementer with diagnosis and guidance appended to prompt |
| 2nd | TaskStop(task_id) + re-dispatch with broader context (full prior output summary) |
| Escalation | Write escalation-{task_id}.json to repo root, mark task as skipped, move to next task |

Escalation file format (from design doc):
```json
{
  "task_id": "A3",
  "issue": "description of what went wrong",
  "attempts": 2,
  "last_output_snippet": "last 50 lines of output",
  "timestamp": "ISO8601"
}
```
