# A4: Create hooks.json and wire into marketplace.json

**Avoid:** Don't add hooks to the `claude-caliper-tooling` plugin — it only includes codebase-review and skill-eval, which never trigger the design approval flow. Adding hooks there would fire the PermissionRequest hook on Edit/Write operations with no possibility of a sentinel existing, adding unnecessary overhead.

**Avoid:** Don't use a relative path for the command in hooks.json — use `${CLAUDE_PLUGIN_ROOT}/hooks/...` so the plugin loader resolves the correct absolute path at runtime regardless of CWD. If `CLAUDE_PLUGIN_ROOT` is not set, the hook commands will fail to resolve and Claude Code will fall back to normal permission behavior (safe degradation).

## Steps

### Step 1: Create hooks/hooks.json

Create `hooks/hooks.json` with the exact content from the design doc:

```json
{
  "PostToolUse": [{
    "matcher": "AskUserQuestion",
    "hooks": [{
      "type": "command",
      "command": "${CLAUDE_PLUGIN_ROOT}/hooks/post-tool-use-design-approval.sh"
    }]
  }],
  "PermissionRequest": [{
    "matcher": "Edit|Write",
    "hooks": [{
      "type": "command",
      "command": "${CLAUDE_PLUGIN_ROOT}/hooks/permission-request-accept-edits.sh"
    }]
  }]
}
```

Validate JSON is well-formed: `jq . hooks/hooks.json`

### Step 2: Add hooks field to marketplace.json

Read `.claude-plugin/marketplace.json`. Add `"hooks": "./hooks/hooks.json"` to the first plugin entry (`claude-caliper`) and the second plugin entry (`claude-caliper-workflow`). Do NOT add it to the third entry (`claude-caliper-tooling`).

The field should be added as a sibling to `"skills"` in each plugin object. After editing, the first plugin entry should look like:

```json
{
  "name": "claude-caliper",
  "description": "claude-caliper bundle",
  "version": "1.6.0",
  "source": "./",
  "author": { "name": "Nikhil Sitaram", "email": "nikhil5890@gmail.com" },
  "strict": false,
  "hooks": "./hooks/hooks.json",
  "skills": [
    "./skills/design",
    ...
  ]
}
```

Same pattern for `claude-caliper-workflow`. Verify with:

```bash
jq -e '.plugins[0].hooks == "./hooks/hooks.json"' .claude-plugin/marketplace.json
jq -e '.plugins[1].hooks == "./hooks/hooks.json"' .claude-plugin/marketplace.json
jq -e '.plugins[2].hooks == null' .claude-plugin/marketplace.json
```

### Step 3: Run all hook tests to confirm nothing broke

```bash
bash tests/hooks/test_post_tool_use.sh && bash tests/hooks/test_permission_request.sh
```
