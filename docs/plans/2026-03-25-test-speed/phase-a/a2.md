# A2: Create gh mock stub fixture

**Avoid:** Do not make the mock overly complex — it only needs to handle `pr list` subcommands. Do not attempt to parse `--json` or `--jq` flags; the real `gh pr list --json number --jq 'length'` returns a bare integer, so the mock just echoes `GH_MOCK_PR_COUNT`. Do not create the mock as a bash function — validate-plan invokes `gh` as a subprocess, so it must be an executable file on PATH.

## Steps

### Step 1: Create the mock script

Create `tests/validate-plan/fixtures/gh-mock.sh`:

```bash
#!/usr/bin/env bash
if [[ "${1:-}" == "pr" && "${2:-}" == "list" ]]; then
  echo "${GH_MOCK_PR_COUNT:-0}"
  exit 0
fi
exit 1
```

The script checks if the first two positional args are `pr` and `list`. All other arguments (`--base`, `--head`, `--state`, `--json`, `--jq`) are ignored — the mock doesn't need them. It reads `GH_MOCK_PR_COUNT` (default `0`) and prints it as a bare integer on stdout, mimicking the `--jq 'length'` output format.

### Step 2: Make it executable

```bash
chmod +x tests/validate-plan/fixtures/gh-mock.sh
```

### Step 3: Verify the mock works

```bash
GH_MOCK_PR_COUNT=0 bash tests/validate-plan/fixtures/gh-mock.sh pr list --base main --head test --json number --jq 'length'
```

Expected output: `0`, exit code 0.

```bash
GH_MOCK_PR_COUNT=1 bash tests/validate-plan/fixtures/gh-mock.sh pr list --base main --head test --json number --jq 'length'
```

Expected output: `1`, exit code 0.

```bash
bash tests/validate-plan/fixtures/gh-mock.sh version
```

Expected: exit code 1 (unrecognized command).

### Step 4: Commit

```bash
git add tests/validate-plan/fixtures/gh-mock.sh
git commit -m "test: add gh mock stub fixture for offline workflow tests"
```
