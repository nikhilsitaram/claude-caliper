# A2: Create safe-commands.txt and PreToolUse hook script

**Avoid:** Don't use `python` or `node` for JSON parsing -- stick to `jq` for consistency with the rest of the repo's hook scripts (`hooks/post-tool-use-design-approval.sh`, `hooks/permission-request-accept-edits.sh`). The hook must work on macOS and Linux with only bash and jq.

**Avoid:** Don't attempt to parse heredocs, process substitution `<()`, or deeply nested subshells. When parsing is ambiguous, fall through to auto mode (return nothing). False negatives (prompting for a safe command) are acceptable; false positives (auto-approving an unsafe command) are security violations.

**Avoid:** Don't write the allow JSON to stderr or anywhere except stdout. The PreToolUse hook protocol expects JSON on stdout for permission decisions. Diagnostics go to stderr.

### Step 2: Write the hook script

Create `hooks/pretooluse-safe-commands.sh`. The script:

1. Reads JSON from stdin (`tool_name`, `tool_input.command`)
2. Exits immediately if `tool_name` is not `Bash`
3. Reads the safe commands list from `$SAFE_COMMANDS_FILE` (env var, for testability) or falls back to `$SCRIPT_DIR/safe-commands.txt`
4. Extracts the command string from `tool_input.command`
5. Parses command words:
   - Split on `&&`, `;`, `|` (respecting quoted strings)
   - For each segment, strip leading whitespace and variable assignments (`VAR=...` prefix before a command)
   - Extract command word from `$(...)` subshells
   - Take the basename of paths (`./node_modules/.bin/jest` -> `jest`)
   - Cap at 20 command words to bound processing
6. Check each command word against the safe list (exact prefix match on basename)
7. If ALL match: output `{"permissionDecision": "allow"}` to stdout
8. If ANY don't match: append the non-matching command words to the log file (`$CLAUDE_SAFE_CMDS_LOG` env var or `$TMPDIR/claude-safe-cmds-nonmatch.log`) and exit silently (no stdout)

The parsing approach -- use `jq` to extract the command string, then a bash function that iterates character-by-character to respect quote boundaries when splitting on `&&`, `;`, `|`. This avoids the pitfalls of naive `IFS` splitting which breaks on quoted strings containing delimiters.

Alternatively, a simpler approach: use `jq` to extract the command, then use a sed/awk pipeline to split on unquoted delimiters. The key requirement is that `echo "hello && world"` produces only `echo` as a command word, not `echo`, `world`.

Make the script executable: `chmod +x hooks/pretooluse-safe-commands.sh`
