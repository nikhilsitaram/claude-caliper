# A1: Update validate-plan: remove supervision, add file-set overlap and status checks

**Avoid:** Don't check file-set overlap across phases — only within the same phase. Cross-phase overlap is valid because phases execute sequentially. Don't add runtime checks for "all tasks complete before phase marked complete" — that's already enforced by `do_update_status_phase()` review gates.

**Avoid:** Don't just check `create` arrays for overlap (that already exists via `duplicate_create_path`). The new check covers all three arrays: `create`, `modify`, and `test`, and also detects cross-array overlap (e.g., task A1 creates `foo.ts` while task A2 modifies `foo.ts`).

## Steps

### Step 1: Write failing tests for file-set overlap

Create `tests/validate-plan/test_fileset_overlap.sh` with these test cases:
1. Plan with no overlap passes
2. Same file in `create` of two tasks in same phase fails
3. Same file in `modify` of two tasks in same phase fails
4. Same file in `test` of two tasks in same phase fails
5. Cross-array overlap within phase fails (A1 creates, A2 modifies same file)
6. Cross-phase overlap passes (A1 creates in Phase A, B1 modifies in Phase B)

Each test creates a temp directory with plan.json and required phase/task files, then runs `bash scripts/validate-plan --schema`.

### Step 2: Write failing tests for status consistency

Create `tests/validate-plan/test_status_consistency.sh` with test cases:
1. Valid plan with matching task ID prefixes passes
2. Task with mismatched prefix (B1 in Phase A) fails
3. Out-of-order phases (B before A) fails
4. Valid multi-phase plan (A then B) passes

### Step 3: Update supervision schema tests

Update `tests/validate-plan/test_supervision_schema.sh` — replace the old tests that validated supervision field acceptance with two simple tests:
1. Plan without supervision field passes
2. Plan with legacy supervision field still passes (field is ignored, not rejected)

### Step 4: Verify new tests fail (red phase)

```bash
bash tests/validate-plan/test_fileset_overlap.sh
bash tests/validate-plan/test_status_consistency.sh
```

Tests 2-5 of fileset_overlap and tests 2-3 of status_consistency should fail.

### Step 5: Remove supervision validation from do_schema()

In `scripts/validate-plan`, remove the entire supervision validation block from `do_schema()` — the section starting with `local has_supervision` (around line 88) through the closing `fi` (around line 123). This block validates supervision object structure, field types, and positive integers. After removal, plans with a `supervision` field simply pass (the field is ignored by jq since no code reads it).

### Step 6: Add file-set overlap detection to do_schema()

In `do_schema()`, after the per-task loop for each phase (after the task iteration `done`, before the phase loop's `done`), add a within-phase overlap check:

```bash
    local -A phase_file_owners=()
    for ((t=0; t<task_count; t++)); do
      local tid
      tid=$(echo "$json" | jq -r ".phases[$p].tasks[$t].id // empty")
      for arr in create modify test; do
        local arr_count
        arr_count=$(echo "$json" | jq "(.phases[$p].tasks[$t].files.$arr // []) | length")
        for ((f=0; f<arr_count; f++)); do
          local fpath
          fpath=$(echo "$json" | jq -r ".phases[$p].tasks[$t].files.$arr[$f]")
          if [[ -n "${phase_file_owners[$fpath]+x}" ]]; then
            err "ERROR: fileset_overlap: file '$fpath' claimed by tasks ${phase_file_owners[$fpath]} and $tid in phase $letter"
          else
            phase_file_owners[$fpath]="$tid"
          fi
        done
      done
    done
```

This uses an associative array scoped per phase. Each file path is tracked to its owning task ID. If a second task claims the same path (from any of create/modify/test), an error is emitted.

### Step 7: Add task ID prefix check to do_schema()

Inside the per-task loop, after extracting and validating the task ID, add:

```bash
      if [[ -n "$id" && -n "$letter" ]]; then
        local id_prefix="${id:0:1}"
        if [[ "$id_prefix" != "$letter" ]]; then
          err "ERROR: task_id_phase_mismatch: task $id has prefix '$id_prefix' but belongs to phase $letter"
        fi
      fi
```

### Step 8: Add alphabetical phase ordering check to do_schema()

Before the phase loop, add `local prev_phase_letter=""`. Inside the phase loop, after extracting and validating the letter, add:

```bash
      if [[ -n "$letter" && -n "$prev_phase_letter" ]]; then
        if [[ "$letter" < "$prev_phase_letter" || "$letter" == "$prev_phase_letter" ]]; then
          err "ERROR: phase_order: phase $letter must come after $prev_phase_letter (alphabetical order required)"
        fi
      fi
      prev_phase_letter="$letter"
```

Note: The duplicate_phase_letter check already catches equality, so the `==` branch is defense-in-depth.
