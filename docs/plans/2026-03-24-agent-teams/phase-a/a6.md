# A6: Update task-reviewer-prompt.md for teammate model

**Avoid:** Don't change the 6-point checklist or the output format — those are working well. The changes are structural: how the reviewer is dispatched (teammate, not subagent of phase dispatcher) and what context it receives.

## Steps

### Step 1: Read current task-reviewer-prompt.md

Read `skills/orchestrate/task-reviewer-prompt.md` end-to-end. Note:
- Currently dispatched as `Agent tool` by phase dispatcher
- Receives `{IMPLEMENTER_REPORT}` from the dispatcher (who read the implementer's output)
- Git range is within a shared worktree (`{BASE_SHA}..{HEAD_SHA}`)
- No reference to per-task completion files

### Step 2: Update the header and dispatch model

Change from `Agent tool` to teammate spawn. The reviewer teammate is spawned by the lead (orchestrator), not by a phase dispatcher.

Update the header comment to: "Dispatch a reviewer teammate to evaluate a single task's implementation. Spawned by the lead when an implementer teammate finishes."

Add a variables block:
- `{TASK_ID}` — the task ID
- `{TASK_SPEC}` — task metadata + prose combined
- `{TASK_COMPLETION_FILE}` — path to implementer's completion notes (`{PHASE_DIR}/{task_id_lower}-completion.md`)
- `{REPO_PATH}` — implementer teammate's worktree path
- `{BASE_SHA}` — SHA before task started
- `{HEAD_SHA}` — SHA after task completed

### Step 3: Replace Implementer Report section

Remove the `{IMPLEMENTER_REPORT}` section. Replace with:

```text
  ## Implementer Completion Notes

  Read {TASK_COMPLETION_FILE} for the implementer's self-reported summary,
  deviations, files changed, and test results.

  Do not trust the notes at face value. Verify every claim by reading
  actual code. The notes are a starting point, not evidence.
```

### Step 4: Update git range context

The reviewer now gets the task's branch diff. Update the Git Range section to use the same variables but with the note that `{REPO_PATH}` is the implementer teammate's worktree path and `{BASE_SHA}/{HEAD_SHA}` bracket the task's work.

### Step 5: Verify 6-point checklist and output format unchanged

Read through sections 1-6 (Spec Fidelity, TDD Discipline, Test Quality, Code Correctness, Security, Simplicity) and the output format (Issues Found, Assessment, Review Summary). Confirm no changes needed.
