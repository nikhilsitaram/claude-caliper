# A11: Register skill and bump plugin version

**Avoid:** Don't change only one plugin entry — all three plugins in `marketplace.json` share the same version and must be bumped together. Don't add the caliper-settings skill to the `claude-caliper-tooling` plugin — it's a workflow skill, not a standalone tool.

## Steps

### Step 1: Add caliper-settings to skill arrays in `.claude-plugin/marketplace.json`

Add `"./skills/caliper-settings"` to the `skills` array of:
- `claude-caliper` (the bundle plugin) — add after `"./skills/skill-eval"`
- `claude-caliper-workflow` (the workflow plugin) — add after `"./skills/pr-merge"`

Do NOT add it to `claude-caliper-tooling` (standalone tools only).

### Step 2: Bump version in all three plugins

Update the `version` field in all three plugin entries from `"1.18.2"` to `"1.19.0"`:

```bash
jq '(.plugins[].version) = "1.19.0"' .claude-plugin/marketplace.json > tmp.json && mv tmp.json .claude-plugin/marketplace.json
```

Then add the skill path to the two plugin entries:

```bash
jq '(.plugins[0].skills) += ["./skills/caliper-settings"] | (.plugins[1].skills) += ["./skills/caliper-settings"]' .claude-plugin/marketplace.json > tmp.json && mv tmp.json .claude-plugin/marketplace.json
```

### Step 3: Verify

```bash
jq -r '.plugins[].version' .claude-plugin/marketplace.json
```

Expected — three lines, all `1.19.0`:

```text
1.19.0
1.19.0
1.19.0
```

```bash
jq '.plugins[0].skills[]' .claude-plugin/marketplace.json | grep -q 'caliper-settings'
jq '.plugins[1].skills[]' .claude-plugin/marketplace.json | grep -q 'caliper-settings'
```

Both should exit 0. Verify `plugins[2]` (tooling) does NOT have it:

```bash
jq '.plugins[2].skills[]' .claude-plugin/marketplace.json | grep -q 'caliper-settings' && echo "ERROR: should not be in tooling plugin" && exit 1 || echo "OK: not in tooling plugin"
```

Commit: `chore: register caliper-settings skill, bump version to 1.19.0`
