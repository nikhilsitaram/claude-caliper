# A3: Write test suite for caliper-settings script

**Avoid:** Don't hardcode setting names or default values in test assertions — read them from `defaults.json` using jq so the tests stay correct if settings are added or defaults change. This follows the project's test convention: "never hardcode configurable values in tests — import constants/defaults from source and derive test data from them." Don't use a different test harness — follow the `assert_pass`/`assert_fail` pattern from existing test files like `tests/validate-plan/test_schema.sh`.

## Steps

### Step 1: Create test directory and file

```bash
mkdir -p tests/caliper-settings
```

Create `tests/caliper-settings/test_caliper_settings.sh` with the standard test boilerplate:

```bash
#!/usr/bin/env bash
set -euo pipefail

SCRIPT_DIR="$(cd "$(dirname "$0")" && pwd)"
REPO_ROOT="$(cd "$SCRIPT_DIR/../.." && pwd)"
SCRIPT="$REPO_ROOT/scripts/caliper-settings"
DEFAULTS="$REPO_ROOT/defaults.json"
PASS=0
FAIL=0

assert_pass() {
  local desc="$1"; shift
  if "$@" > /dev/null 2>&1; then
    echo "PASS: $desc"
    ((PASS++)) || true
  else
    echo "FAIL: $desc"
    ((FAIL++)) || true
  fi
}

assert_fail() {
  local desc="$1"; shift
  local expected_error="$1"; shift
  local output
  if output=$("$@" 2>&1); then
    echo "FAIL: $desc (expected failure, got success)"
    ((FAIL++)) || true
  elif echo "$output" | grep -q "$expected_error"; then
    echo "PASS: $desc"
    ((PASS++)) || true
  else
    echo "FAIL: $desc (expected '$expected_error' in output, got: $output)"
    ((FAIL++)) || true
  fi
}

assert_output() {
  local desc="$1"; shift
  local expected="$1"; shift
  local actual
  actual=$("$@" 2>&1)
  if [[ "$actual" == "$expected" ]]; then
    echo "PASS: $desc"
    ((PASS++)) || true
  else
    echo "FAIL: $desc (expected '$expected', got '$actual')"
    ((FAIL++)) || true
  fi
}
```

### Step 2: Write test cases for `get`

Each test creates a fresh `TMPDIR` for `CLAUDE_PLUGIN_DATA`.

**Test: get returns default when no user override exists**
Read the first key and its default from defaults.json dynamically:
```bash
first_key=$(jq -r 'keys[0]' "$DEFAULTS")
first_default=$(jq -r --arg k "$first_key" '.[$k].default' "$DEFAULTS")
```
Assert that `CLAUDE_PLUGIN_ROOT=$REPO_ROOT CLAUDE_PLUGIN_DATA=$TMPDIR bash "$SCRIPT" get "$first_key"` outputs `$first_default`.

**Test: get returns user override when set**
Set a value first, then get it. For a bool key, set to `true`, verify get returns `true`.

**Test: get with unknown key exits 1**
`assert_fail "unknown key" "Unknown setting" env CLAUDE_PLUGIN_ROOT=$REPO_ROOT CLAUDE_PLUGIN_DATA=$TMPDIR bash "$SCRIPT" get nonexistent_key`

### Step 3: Write test cases for `set`

**Test: set valid bool value**
Set `skip_tests` to `true`, read settings.json, verify the value is JSON `true` (boolean, not string).

**Test: set invalid bool value rejects**
`assert_fail "invalid bool" "expected bool" env ... bash "$SCRIPT" set skip_tests banana`

**Test: set valid enum value**
Read an enum key from defaults.json, get its first valid value, set it, verify.

**Test: set invalid enum value rejects**
`assert_fail "invalid enum" "expected one of" env ... bash "$SCRIPT" set review_mode invalid_mode`

**Test: set valid int value**
`bash "$SCRIPT" set review_wait_minutes 15`, verify get returns `15`.

**Test: set invalid int value rejects**
`assert_fail "invalid int" "expected int" env ... bash "$SCRIPT" set review_wait_minutes abc`

**Test: set unknown key rejects**
`assert_fail "unknown key on set" "Unknown setting" env ... bash "$SCRIPT" set bad_key value`

### Step 4: Write test cases for `reset`

**Test: reset removes single key override**
Set a value, reset it, verify get returns the default.

**Test: reset all removes all overrides**
Set two values, reset with no args, verify both return defaults.

**Test: reset when no settings.json is a no-op**
`assert_pass "reset with no settings file" env ... bash "$SCRIPT" reset`

### Step 5: Write test cases for `list`

**Test: list shows all settings**
Capture output of `list`, verify it contains all 8 keys from defaults.json. Read keys dynamically: `jq -r 'keys[]' "$DEFAULTS"`, loop and check each key appears in the output.

### Step 6: Write error handling tests

**Test: missing CLAUDE_PLUGIN_ROOT**
`assert_fail "missing CLAUDE_PLUGIN_ROOT" "CLAUDE_PLUGIN_ROOT not set" env -u CLAUDE_PLUGIN_ROOT bash "$SCRIPT" list`

**Test: missing CLAUDE_PLUGIN_DATA**
`assert_fail "missing CLAUDE_PLUGIN_DATA" "CLAUDE_PLUGIN_DATA not set" env CLAUDE_PLUGIN_ROOT=$REPO_ROOT -u CLAUDE_PLUGIN_DATA bash "$SCRIPT" list`

**Test: corrupted settings.json falls back to defaults**
Write invalid JSON to `$TMPDIR/settings.json`, verify `get` still returns the default (with a warning to stderr).

### Step 7: Add summary footer and run

```bash
echo ""
echo "Results: $PASS passed, $FAIL failed"
if [[ $FAIL -gt 0 ]]; then exit 1; fi
```

Run the test suite:

```bash
bash tests/caliper-settings/test_caliper_settings.sh
```

All tests should pass.
