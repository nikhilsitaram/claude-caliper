# A2: Implement scripts/caliper-settings bash script

**Avoid:** Don't hardcode defaults in the script — always read from `defaults.json`. The script is a generic engine; all setting knowledge lives in the schema. Don't use `python` or `node` — this repo's scripts are pure bash+jq (see `scripts/validate-plan` for precedent). Don't create `settings.json` eagerly on first run — only write it when `set` is called. A missing `settings.json` means "all defaults," which is the expected first-run state.

## Steps

### Step 1: Create `scripts/caliper-settings`

Create the file with `#!/usr/bin/env bash` and `set -euo pipefail`. Make it executable (`chmod +x`).

The script structure:

```bash
#!/usr/bin/env bash
set -euo pipefail

DEFAULTS_FILE="${CLAUDE_PLUGIN_ROOT:?CLAUDE_PLUGIN_ROOT not set — cannot locate defaults.json}/defaults.json"
SETTINGS_FILE="${CLAUDE_PLUGIN_DATA:?CLAUDE_PLUGIN_DATA not set — cannot locate settings storage}/settings.json"

# Validate defaults.json exists
if [[ ! -f "$DEFAULTS_FILE" ]]; then
  echo "ERROR: defaults.json not found at $DEFAULTS_FILE" >&2
  exit 1
fi

# Load defaults
DEFAULTS=$(cat "$DEFAULTS_FILE")

# Load user settings (empty object if missing or corrupted)
SETTINGS="{}"
if [[ -f "$SETTINGS_FILE" ]]; then
  if ! SETTINGS=$(jq '.' "$SETTINGS_FILE" 2>/dev/null); then
    echo "WARNING: settings.json is invalid JSON, using defaults" >&2
    SETTINGS="{}"
  fi
fi
```

#### Subcommand: `get <key>`

1. Validate the key exists in defaults: `echo "$DEFAULTS" | jq -e --arg k "$key" '.[$k]' > /dev/null 2>&1` — if not, exit 1 with "Unknown setting: $key. Valid settings: $(echo "$DEFAULTS" | jq -r 'keys | join(", ")')"
2. Check user settings first: `user_val=$(echo "$SETTINGS" | jq -r --arg k "$key" '.[$k] // empty')`
3. If user_val is non-empty, print it. Otherwise print the default: `echo "$DEFAULTS" | jq -r --arg k "$key" '.[$k].default'`
4. For bool defaults, `jq -r` prints `true`/`false` (not `"true"`). For int defaults, it prints the number. For enum defaults, the string value.

#### Subcommand: `set <key> <value>`

1. Validate key exists (same as get)
2. Read the type: `type=$(echo "$DEFAULTS" | jq -r --arg k "$key" '.[$k].type')`
3. Validate value against type:
   - `bool`: value must be `true` or `false` (case-sensitive). Error: "Invalid value for $key: expected bool (true/false), got '$value'"
   - `enum`: value must be in the `values` array. Error: "Invalid value for $key: expected one of [$(values joined)], got '$value'"
   - `int`: value must match `^[0-9]+$`. Error: "Invalid value for $key: expected int, got '$value'"
4. Write to settings.json: if file exists, merge; if not, create. Use jq to handle type-correct JSON encoding:
   - `bool`: `jq --arg k "$key" --argjson v "$value" '.[$k] = $v'` (argjson for true/false as JSON booleans)
   - `int`: `jq --arg k "$key" --argjson v "$value" '.[$k] = $v'` (argjson for number)
   - `enum`: `jq --arg k "$key" --arg v "$value" '.[$k] = $v'` (arg for string)
5. `mkdir -p "$(dirname "$SETTINGS_FILE")"` before writing (in case CLAUDE_PLUGIN_DATA dir doesn't exist yet)

#### Subcommand: `reset [key]`

1. If key provided: validate it exists, then remove from settings.json: `jq --arg k "$key" 'del(.[$k])'`
2. If no key: remove all user overrides — write `{}` to settings.json (or delete the file)
3. If settings.json doesn't exist, this is a no-op (already at defaults)

#### Subcommand: `list`

Print a formatted table with columns: Key, Current Value, Default, Description.

```bash
echo ""
printf "%-25s %-20s %-20s %s\n" "KEY" "CURRENT" "DEFAULT" "DESCRIPTION"
printf "%-25s %-20s %-20s %s\n" "---" "-------" "-------" "-----------"
for key in $(echo "$DEFAULTS" | jq -r 'keys[]'); do
  default=$(echo "$DEFAULTS" | jq -r --arg k "$key" '.[$k].default')
  description=$(echo "$DEFAULTS" | jq -r --arg k "$key" '.[$k].description')
  user_val=$(echo "$SETTINGS" | jq -r --arg k "$key" '.[$k] // empty')
  current="${user_val:-$default}"
  printf "%-25s %-20s %-20s %s\n" "$key" "$current" "$default" "$description"
done
echo ""
```

#### Usage / no subcommand

If no args or unrecognized subcommand, print usage and exit 1:

```text
Usage: caliper-settings <command> [args]

Commands:
  get <key>           Get the current value for a setting
  set <key> <value>   Set a user override for a setting
  reset [key]         Reset a setting to default (or all settings if no key)
  list                Show all settings with current and default values
```
