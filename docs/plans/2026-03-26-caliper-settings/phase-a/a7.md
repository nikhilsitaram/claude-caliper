# A7: Integrate merge_strategy into pr-merge

**Avoid:** Don't change the integration branch auto-detection (`IS_INTEGRATION=true` always uses rebase). Don't change the `--rebase` flag behavior. Phase PRs (base is `integrate/*`) always use `--squash` regardless of settings. The setting only applies to standard single-phase PRs where no flag or auto-detection overrides.

## Steps

### Step 1: Add merge_strategy fallback to Step 2 (Merge) in `skills/pr-merge/SKILL.md`

Find the current merge strategy section:

```markdown
**Merge strategy:**
- Default: `gh pr merge $PR_NUMBER --squash`
- Integration branches (`IS_INTEGRATION=true`): `gh pr merge $PR_NUMBER --rebase` — auto-detected, no flag needed
- Explicit `--rebase` flag overrides default for any branch
```

Replace with:

```markdown
**Merge strategy:**
- Integration branches (`IS_INTEGRATION=true`): `gh pr merge $PR_NUMBER --rebase` — auto-detected, no flag needed
- Phase PRs (base is `integrate/*`): `gh pr merge $PR_NUMBER --squash` — auto-detected, no flag needed
- Explicit `--rebase` flag overrides for any non-auto-detected branch
- Otherwise: check `${CLAUDE_PLUGIN_ROOT}/scripts/caliper-settings get merge_strategy` — use the returned value (`squash` or `rebase`) as the merge method
```

### Step 2: Verify

```bash
grep -q 'caliper-settings get merge_strategy' skills/pr-merge/SKILL.md
```
