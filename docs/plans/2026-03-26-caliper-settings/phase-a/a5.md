# A5: Integrate skip_tests into pr-create

**Avoid:** Don't restructure Step 4 or add a new step — insert the settings fallback as a one-liner within the existing Step 4. Don't change the `--skip-tests`/`-T` flag behavior — CLI flags remain tier 1 and always override.

## Steps

### Step 1: Add settings fallback to Step 4 (Run Tests) in `skills/pr-create/SKILL.md`

Find the current Step 4 text:

```markdown
### Step 4: Run Tests

Auto-detect the project's test runner and run tests. If tests fail, stop and help fix. If no tests found, note and continue.

Skip with `--skip-tests` or `-T`.
```

Replace with:

```markdown
### Step 4: Run Tests

Auto-detect the project's test runner and run tests. If tests fail, stop and help fix. If no tests found, note and continue.

Skip with `--skip-tests` or `-T`. If neither flag was passed, check `${CLAUDE_PLUGIN_ROOT}/scripts/caliper-settings get skip_tests` — if it returns `true`, skip tests.
```

### Step 2: Verify

```bash
grep -q 'caliper-settings get skip_tests' skills/pr-create/SKILL.md
```
