# A8: Integrate workflow, execution_mode, and re_review_threshold into design

**Avoid:** Don't remove the existing Q1/Q2 prompt options — settings provide defaults, but the user still picks via AskUserQuestion. The setting value determines which option is pre-selected/marked as default, not the available choices. Don't change the re-review loop flow — only replace the hardcoded `5` threshold with a dynamic lookup.

## Steps

### Step 1: Add workflow default to Step 7 Q1 in `skills/design/SKILL.md`

Find the Q1 section:

```markdown
    **Q1 — Workflow** (header: "Workflow"):
    - **Create PR** (default) — Orchestrate → pr-create
    - **Merge PR** — Orchestrate → pr-create → pr-review → pr-merge
    - **Plan only** — Stop after plan is reviewed
```

Replace with:

```markdown
    **Q1 — Workflow** (header: "Workflow"):
    Check `${CLAUDE_PLUGIN_ROOT}/scripts/caliper-settings get workflow` for the user's preferred default. Mark the matching option with "(default)":
    - **Create PR** — Orchestrate → pr-create
    - **Merge PR** — Orchestrate → pr-create → pr-review → pr-merge
    - **Plan only** — Stop after plan is reviewed
```

### Step 2: Add execution_mode default to Step 7 Q2 in `skills/design/SKILL.md`

Find the Q2 section header and recommendation logic:

```markdown
    **Q2 — Execution mode** (header: "Exec mode"):
    Recommend based on design complexity:
    - ≤10 tasks AND single phase → recommend `Subagents`
    - >10 tasks OR multi-phase → recommend `Agent teams`

    Mark the recommended option with "(Recommended)" in its label.
```

Replace with:

```markdown
    **Q2 — Execution mode** (header: "Exec mode"):
    Check `${CLAUDE_PLUGIN_ROOT}/scripts/caliper-settings get execution_mode` for the user's preferred default. Recommend based on design complexity (these override the setting when they differ):
    - ≤10 tasks AND single phase → recommend `Subagents`
    - >10 tasks OR multi-phase → recommend `Agent teams`

    Mark the recommended option with "(Recommended)" in its label. If the user's setting differs from the complexity recommendation, note both in the option labels.
```

### Step 3: Add re_review_threshold to design-review dispatch (Step 9) in `skills/design/SKILL.md`

Find the re-review logic in the design-review dispatch section. The current text references `>5 actionable issues` in the plan-review dispatch section:

```markdown
If >5 actionable issues, fix and re-dispatch reviewer (max 3 iterations, then escalate to user).
```

Replace with:

```markdown
Read the threshold: `${CLAUDE_PLUGIN_ROOT}/scripts/caliper-settings get re_review_threshold`. If actionable issues exceed this threshold, fix and re-dispatch reviewer (max 3 iterations, then escalate to user).
```

Also find the equivalent in the design-review section (step 84 area). The design skill text at the end of step 9 says:

```markdown
**Re-review gate:** If the reviewer finds more than 5 issues, after all fixes, dispatch a fresh reviewer with the same full scope to confirm clean.
```

If this text appears inline in the design-review dispatch block, replace `more than 5 issues` with `more issues than \`${CLAUDE_PLUGIN_ROOT}/scripts/caliper-settings get re_review_threshold\``.
